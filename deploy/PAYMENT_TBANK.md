# Платежи T-Bank: демо-терминал и боевой режим

В документе «Протестируйте платежи» (SPRUT RUN STORE) описаны два варианта: **тестовый (демо-терминал)** и **рабочий (боевой терминал)**. Ниже — пошагово, как сначала протестировать через демо, затем переключиться на приём реальных платежей.

---

## Этап 1: Тестирование через демо-терминал

Демо-терминал позволяет пройти полный цикл: создание платежа в T-Bank, форма оплаты, тестовые карты, возврат на сайт и (при настроенном вебхуке) обновление статуса заказа. Деньги не списываются.

### 1.1. Данные демо-терминала (из документа)

| Параметр   | Значение            |
|-----------|----------------------|
| Терминал  | `1771878373032DEMO`  |
| Пароль    | `J9ttMgcoh^Yp9AtX`   |

Эти значения уже прописаны в `config/payment.php` для режима демо.

### 1.2. Настройка `.env` для демо

Добавьте или измените в `.env`:

```env
# Отключаем локальную имитацию (редирект на «Оплата прошла успешно» без API)
PAYMENT_TEST_MODE=false

# Включаем демо-терминал T-Bank из документа
TBANK_USE_DEMO_TERMINAL=true
```

Пароль демо по умолчанию задан в конфиге. Если в вашей копии документа другой пароль, укажите его явно:

```env
TBANK_DEMO_PASSWORD="J9ttMgcoh^Yp9AtX"
```

На Windows пароль с символом `^` лучше указывать в двойных кавычках.

**Важно:** `APP_URL` должен соответствовать адресу, с которого вы заходите на сайт (например, `http://localhost:8000` или `https://ваш-тестовый-домен.ru`). От него формируются ссылки возврата после оплаты и (при необходимости) вебхук.

### 1.3. URL возврата и вебхук (для демо)

- **SuccessURL / FailURL** — если не заданы, используются ссылки на страницу подтверждения заказа (`/order/{id}/confirmation`), сформированные из `APP_URL`. Для локальной разработки достаточно корректного `APP_URL`.
- **Вебхук (NotificationURL)** — нужен, чтобы T-Bank присылал уведомления об оплате и заказ автоматически переходил в статус «Оплачен».  
  Для теста с `localhost` T-Bank не сможет вызвать ваш вебхук (нет доступа из интернета). Проверить вебхук можно:
  - на тестовом сервере с белым IP и HTTPS, либо
  - временно пропустить: после успешной оплаты на форме T-Bank вас вернёт на SuccessURL, но статус «Оплачен» в БД появится только после обработки вебхука (или ручного обновления).

При необходимости задайте в `.env` (для тестового сервера с HTTPS):

```env
TBANK_NOTIFICATION_URL=https://ваш-тестовый-домен.ru/payment/webhook
TBANK_SUCCESS_URL=https://ваш-тестовый-домен.ru/order/{orderId}/confirmation
TBANK_FAIL_URL=https://ваш-тестовый-домен.ru/order/{orderId}/confirmation
```

В коде SuccessURL/FailURL подставляются из маршрута с конкретным `order`, поэтому при использовании переменных окружения убедитесь, что в личном кабинете T-Bank или в коде не перезаписывается подстановка `{orderId}`.

### 1.4. Что проверить на демо

1. Оформление заказа в магазине → редирект на форму оплаты T-Bank (не на локальную страницу «Оплата прошла успешно (тестовый режим)»).
2. На форме T-Bank — ввод тестовой карты (номера тестовых карт см. в документации T-Bank / в личном кабинете).
3. Успешная оплата → возврат на сайт по SuccessURL, на странице заказа отображается успешное сообщение.
4. Неуспешная оплата / отмена → возврат по FailURL.
5. Если вебхук настроен и доступен из интернета — после успешной оплаты у заказа проставляется «Оплачен» и `paid_at` в БД (можно проверить в админке или в БД).

После этого можно переходить к настройке боевого терминала.

## Этап 2: Переход на рабочий (боевой) терминал

Когда тесты с демо-терминалом пройдены, переключите проект на приём реальных платежей через рабочий терминал из Т-Бизнес.

### 2.1. Где взять данные боевого терминала

- Войдите в [Т-Бизнес](https://www.tbank.ru/business/) (интернет-эквайринг).
- Откройте настройки терминала (или раздел с данными для интеграции).
- Скопируйте:
  - **Идентификатор терминала (Terminal Key)** — в `.env` это `TBANK_TERMINAL_KEY`;
  - **Пароль терминала** — в `.env` это `TBANK_PASSWORD`.

В документе «Протестируйте платежи» рабочий вариант указан рядом с демо — используйте именно данные **рабочего** терминала, не `1771878373032DEMO`.

### 2.2. Настройка `.env` для боевого режима

Отключите демо и укажите боевые данные:

```env
# Режим оплаты через API T-Bank (не локальная имитация)
PAYMENT_TEST_MODE=false

# Отключаем демо-терминал
TBANK_USE_DEMO_TERMINAL=false

# Данные рабочего терминала из Т-Бизнес
TBANK_TERMINAL_KEY=ваш_боевой_терминал
TBANK_PASSWORD=ваш_боевой_пароль
```

Пароль при необходимости тоже укажите в кавычках, если в нём есть спецсимволы.

### 2.3. Обязательные URL для боевого режима

На production обязательно задайте:

```env
APP_URL=https://ваш-домен.ru

# URL, куда T-Bank отправляет уведомления об оплате (должен быть доступен по HTTPS из интернета)
TBANK_NOTIFICATION_URL=https://ваш-домен.ru/payment/webhook
```

SuccessURL и FailURL при пустых значениях в коде формируются автоматически из `APP_URL` и маршрута подтверждения заказа. При необходимости их можно переопределить:

```env
TBANK_SUCCESS_URL=https://ваш-домен.ru/order/{id}/confirmation
TBANK_FAIL_URL=https://ваш-домен.ru/order/{id}/confirmation
```

(В коде подставляется конкретный `order->id`, поэтому при использовании этих переменных убедитесь, что в коде или в ЛК T-Bank не передаётся буквальная строка `{id}`.)

### 2.4. Настройки в личном кабинете T-Bank

- В настройках терминала укажите тот же **NotificationURL**, что и `TBANK_NOTIFICATION_URL` в `.env` (например, `https://ваш-домен.ru/payment/webhook`), если ЛК это позволяет.
- Проверьте, что терминал включён для приёма платежей и при необходимости привяжите нужные способы оплаты (карты, СБП и т.д.).

### 2.5. Чек-лист перед приёмом реальных платежей

- [ ] `PAYMENT_TEST_MODE=false`
- [ ] `TBANK_USE_DEMO_TERMINAL=false` (или переменная удалена)
- [ ] `TBANK_TERMINAL_KEY` и `TBANK_PASSWORD` — от **боевого** терминала из Т-Бизнес
- [ ] `APP_URL` — боевой домен с HTTPS
- [ ] `TBANK_NOTIFICATION_URL` — доступен из интернета по HTTPS, путь `/payment/webhook`
- [ ] Маршрут `POST /payment/webhook` исключён из проверки CSRF (в `bootstrap/app.php` уже добавлено)
- [ ] После тестовой оплаты на боевом терминале проверены: возврат на сайт, обновление статуса заказа и запись в `payments` / `orders.paid_at`

## Краткая сводка переключения

| Режим              | PAYMENT_TEST_MODE | TBANK_USE_DEMO_TERMINAL | TBANK_TERMINAL_KEY / TBANK_PASSWORD      |
|--------------------|-------------------|--------------------------|------------------------------------------|
| Локальная имитация | `true`            | не важно                 | не нужны                                 |
| Демо T-Bank        | `false`           | `true`                   | не нужны (берутся из конфига)            |
| Боевой терминал    | `false`           | `false`                  | данные рабочего терминала из Т-Бизнес    |

После смены переменных достаточно обновить приложение (при необходимости перезапустить воркеры/очереди, если они кэшируют конфиг). Отдельный перезапуск сервера не обязателен — Laravel читает `.env` при каждом запросе.
