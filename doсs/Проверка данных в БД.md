# Проверка данных в базе данных

## Способ 1: Через Laravel Tinker (рекомендуется)

Откройте терминал и выполните:

```bash
php artisan tinker
```

Затем выполните команды для проверки:

```php
// Проверка пользователей
App\Models\User::count();
App\Models\User::all();

// Проверка городов
App\Models\City::count();
App\Models\City::all();

// Проверка партнёров
App\Models\Partner::count();
App\Models\Partner::all();

// Проверка событий
App\Models\Event::count();
App\Models\Event::all();

// Проверка предстоящих событий
App\Models\Event::where('status', 'published')->where('starts_at', '>', now())->get();

// Проверка прошедших событий (архив)
App\Models\Event::where('status', 'published')->where('starts_at', '<=', now())->get();

// Проверка связей событий и партнёров
$event = App\Models\Event::first();
$event->partners;

// Выход из Tinker
exit
```

## Способ 2: Через SQL запросы

Подключитесь к MySQL:

```bash
mysql -u root -p urban_running
```

Или через Laravel:

```bash
php artisan db
```

Выполните SQL запросы:

```sql
-- Количество записей в каждой таблице
SELECT 'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cities', COUNT(*) FROM cities
UNION ALL
SELECT 'partners', COUNT(*) FROM partners
UNION ALL
SELECT 'events', COUNT(*) FROM events
UNION ALL
SELECT 'event_partners', COUNT(*) FROM event_partners;

-- Просмотр пользователей
SELECT id, name, email, created_at FROM users;

-- Просмотр городов
SELECT id, name, country_code, is_active FROM cities;

-- Просмотр партнёров
SELECT id, name, level, is_active FROM partners;

-- Просмотр событий
SELECT id, title, slug, status, starts_at FROM events ORDER BY starts_at;

-- Просмотр связей событий и партнёров
SELECT e.title as event, p.name as partner 
FROM event_partners ep
JOIN events e ON ep.event_id = e.id
JOIN partners p ON ep.partner_id = p.id;
```

## Способ 3: Через веб-интерфейс сайта

Откройте в браузере:

1. **Главная страница** (`http://127.0.0.1:8000/`)
   - Должны отображаться 3 предстоящих события
   - Должны отображаться партнёры

2. **Страница событий** (`http://127.0.0.1:8000/events`)
   - Должны отображаться все 6 событий
   - Статусы "Предстоящее" или "Завершено"

3. **Архив событий** (`http://127.0.0.1:8000/archive`)
   - Должны отображаться 3 прошедших события
   - Группировка по годам

4. **Карточка события** (например, `http://127.0.0.1:8000/events/moscow-spring-2026`)
   - Полная информация о событии
   - Партнёры события (если есть)

5. **Партнёры** (`http://127.0.0.1:8000/partners`)
   - Должны отображаться 2 партнёра

## Способ 4: Быстрая проверка через Artisan команды

Создайте временную команду или используйте:

```bash
# Проверка через модели
php artisan tinker --execute="echo 'Users: ' . App\Models\User::count() . PHP_EOL;"
php artisan tinker --execute="echo 'Cities: ' . App\Models\City::count() . PHP_EOL;"
php artisan tinker --execute="echo 'Partners: ' . App\Models\Partner::count() . PHP_EOL;"
php artisan tinker --execute="echo 'Events: ' . App\Models\Event::count() . PHP_EOL;"
```

## Ожидаемые результаты

После выполнения `php artisan db:seed` должны быть заполнены:

- ✅ **users**: 1 запись
- ✅ **cities**: 3 записи
- ✅ **partners**: 2 записи
- ✅ **events**: 6 записей
- ✅ **event_partners**: несколько связей между событиями и партнёрами

## Проверка конкретных данных

### Проверка событий с партнёрами:

```php
// В Tinker
$event = App\Models\Event::where('slug', 'moscow-spring-2026')->first();
$event->title; // "Весенний забег в Москве 2026"
$event->partners->count(); // должно быть 2
$event->partners->pluck('name'); // список партнёров
```

### Проверка города события:

```php
$event = App\Models\Event::first();
$event->city->name; // название города
```

### Проверка статусов событий:

```php
// Предстоящие
App\Models\Event::where('status', 'published')
    ->where('starts_at', '>', now())
    ->count(); // должно быть 3

// Прошедшие (архив)
App\Models\Event::where('status', 'published')
    ->where('starts_at', '<=', now())
    ->count(); // должно быть 3
```
